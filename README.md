# أرز مارت - Arz-Mart 🌲🛒

متجر إلكتروني متكامل للتجارة الإلكترونية مصمم خصيصاً للسوق اللبناني، يحتوي على واجهة متجر تفاعلية، لوحة تحكم إدارية احترافية، وتطبيق أندرويد مدمج يدعم البصمة الحيوية (Biometrics).

---

## 📂 هيكلية المشروع (Project Architecture)

ينقسم المشروع إلى ثلاثة أقسام رئيسية:
1. **السيرفر الخلفي (`/backend`)**: مبني باستخدام Express.js وقاعدة بيانات SQLite لتخزين المنتجات، المستخدمين، التجار، الطلبيات، والمحادثات.
2. **الواجهة الأمامية (`/frontend`)**: مبنية باستخدام React.js و Vite و Vanilla CSS بتصميم راقٍ يدعم الوضعين المظلم والمضيء واللغتين العربية والإنجليزية.
3. **تطبيق الأندرويد (`/android`)**: تطبيق أصيل (Native Kotlin/Gradle) يقوم بتحميل موقع المتجر مع حماية مدمجة عبر البصمة الحيوية (Biometrics).

## ✨ ميزات واجهة المتجر للزبائن (Storefront Client Features)

تقدم واجهة المتجر تجربة مستخدم فاخرة وسهلة للعملاء وتتضمن:
* **ثنائية اللغة الكاملة**: واجهة مستخدم مترجمة بالكامل للعربية والإنجليزية تدعم اتجاه القراءة من اليمين لليسار (RTL) ومن اليسار لليمين (LTR).
* **دعم العملة المزدوجة (USD / LBP)**: تبديل فوري للعملة المعروضة بين الدولار الأمريكي والليرة اللبنانية بناءً على سعر الصرف المحدد في لوحة الإدارة.
* **تصميم كروت التصنيفات الفاخر**: تظهر التصنيفات على الصفحة الرئيسية على شكل كروت أنيقة ومرتبة (بواقع 4 كروت في السطر على أجهزة الكمبيوتر وبارتفاع 280 بكسل) وتظهر المنتجات داخل كروت التصنيفات المعنية بدلاً من بعثرتها عشوائياً.
* **سلة المشتريات الديناميكية**: إضافة وتعديل كميات المنتجات مع حساب المجموع التلقائي وقيمة التوصيل.
* **إتمام الطلبيات السهل**: إدخال رقم الهاتف والعنوان بالتفصيل واختيار طريقة الدفع.
* **خيارات الدفع المتنوعة**: دعم الدفع نقدًا عند الاستلام (COD) أو الدفع الإلكتروني (في حال تم تفعيله من الإدارة).
* **كوبونات الخصم التلقائية**: إمكانية تطبيق كوبونات الخصم (مثل كود الترحيب للزبائن الجدد `WELCOME10` بخصم 10% تلقائيًا).
* **تتبع الطلبات التاريخية**: صفحة "طلباتي السابقة" للزبائن تتيح لهم تتبع حالة الطلبية (قيد الانتظار، قيد التحضير، تم الشحن، تم التسليم) مع رقم التتبع.
* **محادثة فورية (Live Chat)**: صندوق محادثة مباشر يتيح للزبون التحدث فوراً مع موظفي الإدارة.

## 🛡️ ميزات لوحة تحكم الإدارة (Admin Dashboard Features)

لوحة تحكم كاملة واحترافية مقسمة لعدة أقسام تسهل إدارة المتجر بالكامل:
1. **التقارير المالية والأرباح**:
   * عرض إجمالي المبيعات (يومياً وشهرياً).
   * حساب **الأرباح الصافية الحقيقية** بناءً على (سعر البيع - سعر التكلفة الحقيقي لكل منتج) وليس نسبة مئوية افتراضية.
   * إحصائيات المخزون والقيمة الإجمالية للمنتجات.
2. **إدارة التصنيفات الهرمية (Nested Categories)**:
   * إمكانية إضافة تصنيف رئيسي وبداخله تصنيفات فرعية متعددة المستويات لتسهيل تنظيم المتجر.
3. **إدارة المنتجات (Products Management)**:
   * إضافة، تعديل، وحذف المنتجات مع تحديد الاسم والوصف (بالعربي والإنجليزي)، السعر الحالي، **سعر التكلفة (Cost Price)** لحساب الأرباح، المخزون، وربطها بالتاجر المصنع.
4. **إدارة التجار والموردين (Merchants)**:
   * إضافة وتعديل بيانات التجار لتحديد مصدر كل منتج ومتابعة التوريد.
5. **إدارة الطلبيات والفواتير (Orders Management)**:
   * مراقبة الطلبات الجديدة والطلبات المسلمة.
   * تغيير حالة الطلب وتوليد رقم التتبع.
   * **طباعة الفاتورة الذكية**: زر لطباعة الفاتورة للزبون، وزر آخر لـ **"الطباعة بدون سعر"** لإرسال الفاتورة مع موظف التوصيل دون كشف تكلفة المنتجات.
6. **إدارة الصلاحيات والموظفين (Users & Staff Permissions)**:
   * استعراض المستخدمين وتعيين الموظفين.
   * منح صلاحيات مخصصة لكل موظف للتحكم في أقسام معينة فقط (مثل: المنتجات فقط، الطلبيات فقط، التقارير فقط، الإعدادات، إلخ).
7. **إدارة الكوبونات والخصومات (Coupons)**:
   * إنشاء وتفعيل وحذف كوبونات الخصم وتحديد النسبة المئوية للخصم.

## 📱 تطبيق الأندرويد والتحقق البايومتري (Android & Biometrics)

تم دمج تطبيق أندرويد متكامل يعمل كغلاف (WebView Project) للموقع مع ميزات أمان متقدمة:
* **توجيه الـ Localhost التلقائي**: يقوم التطبيق بتحويل روابط السيرفر المحلي من `localhost` إلى `10.0.2.2` تلقائياً ليعمل بسلاسة داخل محاكي الأندرويد (Android Emulator).
* **حماية البصمة الحيوية (Biometrics)**: عند فتح التطبيق، يطالب المستخدم بتوثيق هويته عبر بصمة الإصبع بشكل آمن بالاعتماد على الـ SDK الأصيل للأندرويد، ويتصل بالمتجر عبر واجهة جافا سكريبت تفاعلية (`window.onBiometricSuccess`).

## 🔒 معايير الأمان المطبقة (Security Standards)

* **تشفير كلمات المرور**: يتم حفظ جميع كلمات المرور في قاعدة البيانات بعد تشفيرها باستخدام مكتبة `bcryptjs`.
* **تعطيل الإكمال التلقائي الحساس**: تم إيقاف ميزة الإكمال التلقائي (`autocomplete="off"`) في الحقول الحساسة مثل تسجيل الدخول لمنع تسرب البيانات.
* **التحقق من الصلاحيات**: السيرفر يتحقق من امتلاك المستخدم للصلاحية المطلوبة قبل السماح له بالوصول لأي وظيفة إدارية (Middleware verification).

## 🖼️ قياسات ومواصفات صور البانرات (Recommended Banner Sizes & Specs)

تدعم لوحة التحكم إدارة عدد غير محدود من البانرات الترويجية المتحركة (Hero Banners) في الصفحة الرئيسية بمرونة تامة.

### المقاسات الموصى بها لأفضل جودة:
* **المقاس المثالي المطابق للتصميم**: **`1200 × 420` بكسل** (أو نسبة عرض إلى ارتفاع مماثلة).
* **مقاس الشاشات الكبيرة والكمبيوتر**: **`1920 × 600` بكسل** للحصول على أعلى درجة وضوح دون زيادة مفرطة في حجم الملف.
* **ملاحظة تقنية**: يتم عرض الصور في المتجر باستخدام خاصية التغطية والتوسيط التلقائي (`background-size: cover` و `background-position: center`) لتتكيف الصورة تلقائياً وبشكل مريح على الهواتف والشاشات المختلفة. يُنصح باستخدام صور ذات توجيه أفقي (Landscape).

## 🚀 طريقة تشغيل المشروع (How to Run)

بسبب عدم توفر بيئة Node.js عالمية على نظام التشغيل، تم تضمين بيئة محليّة خاصة بالمنصة تحت مسار `node-env`. يجب تصدير هذا المسار دائماً قبل تشغيل الأوامر.

### ١. تشغيل السيرفر الخلفي (Backend)
1. افتح مبدل الأوامر (Terminal) في مجلد `backend`.
2. قم بتصدير مسار بيئة Node وتثبيت الحزم ثم التشغيل:
   ```bash
   export PATH=/home/hussein/Desktop/Arz-Mart/node-env/bin:$PATH
   npm install
   npm run start
   ```
   * سيعمل السيرفر على المنفذ `http://localhost:5000`.

### ٢. تشغيل الواجهة الأمامية (Frontend)
1. افتح مبدل الأوامر في مجلد `frontend`.
2. قم بتصدير مسار بيئة Node وتثبيت الحزم ثم التشغيل في وضع التطوير:
   ```bash
   export PATH=/home/hussein/Desktop/Arz-Mart/node-env/bin:$PATH
   npm install
   npm run dev
   ```
   * ستعمل الواجهة على المنفذ `http://localhost:3000`.
3. لبناء الواجهة للإنتاج (Production Build):
   ```bash
   npm run build
   ```
